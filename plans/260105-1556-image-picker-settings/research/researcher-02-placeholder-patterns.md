# Research Report: Image Placeholder Patterns in Shopify Liquid

**Date:** 2026-01-05
**Topic:** Best practices for handling image placeholders with image_picker settings
**Focus:** Conditional rendering, placeholder SVG patterns, responsive images, empty states

## Executive Summary

Shopify provides native placeholder SVG support via `placeholder_svg_tag` filter for image_picker empty states. Best practice: check if image exists using `{% if section.settings.image != blank %}`, render actual image with `image_tag` filter, fallback to `{{ 'image' | placeholder_svg_tag }}`. Use `image_url` for CDN optimization and `image_tag` for responsive markup (srcset, sizes). Avoid conditional checks with `== blank`—use truthiness (`if image`) instead. Add CSS classes to placeholders for styling consistency.

## Key Findings

### 1. Placeholder SVG Tag Filter

**Usage:** `{{ placeholder_name | placeholder_svg_tag: 'optional-class' }}`

- Outputs inline SVG (no network requests, instant rendering)
- Available types: `image`, `product` (6 variants), `collection` (6 variants), `lifestyle` (multiple)
- Supports custom CSS classes: `{{ 'product-1' | placeholder_svg_tag: 'my-class my-class--two' }}`
- Shopify-native, optimized for performance
- Scales to any size without pixelation

### 2. Conditional Rendering Patterns

**Best Practice Pattern:**
```liquid
{% if section.settings.image != blank %}
  {# render actual image #}
{% else %}
  {# render placeholder #}
{% endif %}
```

**Alternative (preferred for truthiness):**
```liquid
{% if section.settings.image %}
  {# render actual image #}
{% else %}
  {# render placeholder #}
{% endif %}
```

Avoid `== blank` syntax when simple truthiness check suffices. Both work; truthiness is more Liquid-idiomatic.

### 3. Image URL & Tag Filters

**`image_url` filter:** Generates optimized image URLs via Shopify CDN
- Auto-selects modern formats (AVIF) when browser supports
- Syntax: `{{ image | image_url: width: 200, height: 200 }}`
- Handles resizing, cropping, padding

**`image_tag` filter:** Generates complete `<img>` markup
- Auto-includes srcset, sizes, alt text
- Syntax: `{{ image | image_url: width: 200 | image_tag: alt: 'description', loading: 'lazy' }}`
- Implements responsive images natively
- Supports lazy loading attribute

### 4. Responsive Image Implementation

**Full responsive pattern:**
```liquid
{{ section.settings.image
   | image_url: width: 1200
   | image_tag:
     alt: section.settings.image.alt,
     loading: 'lazy',
     widths: '360, 480, 600, 800, 1200'
}}
```

**Key attributes:**
- `loading: 'lazy'` for performance
- Auto srcset generation (no manual configuration needed)
- Auto size selection by browser
- Dawn theme pattern: adds loading attribute directly to tags

### 5. Empty State UI Patterns

**Shopify-recommended approach:**
1. Check for image existence
2. Render placeholder SVG with descriptive CSS class
3. Style placeholder with CSS for consistent UX
4. Never show broken images or empty containers

**Schema example:**
```json
{
  "type": "image_picker",
  "id": "hero_image",
  "label": "Hero Image"
}
```

**Liquid rendering:**
```liquid
<div class="hero">
  {% if section.settings.hero_image %}
    {{ section.settings.hero_image
       | image_url: width: 1920
       | image_tag:
         alt: section.settings.hero_image.alt,
         loading: 'lazy',
         sizes: '100vw'
    }}
  {% else %}
    {{ 'lifestyle-1' | placeholder_svg_tag: 'hero__placeholder' }}
  {% endif %}
</div>
```

**CSS styling:**
```css
.hero__placeholder {
  width: 100%;
  height: 500px;
  background-color: var(--color-bg-subtle);
  display: flex;
  align-items: center;
  justify-content: center;
}

.hero__placeholder svg {
  width: 80%;
  height: auto;
  opacity: 0.6;
}
```

### 6. Dawn Theme Patterns

- Conditional rendering in sections/snippets (not templates)
- Loading attribute added directly: `loading="lazy"`
- Width/height specified for accessibility & layout shift prevention
- Srcset/sizes auto-generated by image_tag filter
- SVGs moved to assets folder (recent versions)
- Snippets contain reusable image logic

## Common Patterns Summary

| Pattern | Use Case | Syntax |
|---------|----------|--------|
| Check existence | Conditional rendering | `{% if image %}` |
| Generate URL | CDN optimization | `\| image_url: width: X` |
| Generate tag | Responsive markup | `\| image_tag: alt: '...'` |
| Placeholder SVG | Empty state fallback | `\| placeholder_svg_tag: 'class'` |
| Custom class | Styling placeholders | `'image' \| placeholder_svg_tag: 'class'` |

## Best Practices Checklist

- Always check `if section.settings.image` before rendering
- Use `image_tag` filter (not bare `<img>` tags)
- Add `loading: 'lazy'` for performance
- Provide alt text from image object
- Fallback to `placeholder_svg_tag` when image blank
- Add CSS classes to placeholders for styling
- Never use `image_url` in src attribute alone—chain with `image_tag`
- Specify width/height on placeholders to prevent layout shift
- Test responsive behavior across breakpoints

## Performance Considerations

- `placeholder_svg_tag`: Zero network cost (inline SVG)
- `image_url` + CDN: Auto format selection (AVIF when supported)
- `image_tag`: Generates optimized srcset automatically
- `loading: 'lazy'`: Defers offscreen image loading
- Placeholder SVGs preferred over placeholder images (smaller filesize)
